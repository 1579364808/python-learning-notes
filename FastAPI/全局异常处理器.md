# 全局异常处理器

## 一、为什么需要

没有全局异常处理器时，路由里要自己 try-catch：

```python
# ❌ 每个路由自己兜底，重复代码
@router.get("/detail")
async def get_news_detail(...):
    try:
        news_detail = await crud_get_news_detail(db, news_id)
        if not news_detail:
            raise HTTPException(404, "新闻不存在")
        ...
    except Exception as e:
        return JSONResponse(content={"code": 500, "message": "服务器错误"})
```

有了全局处理器，路由只管正常流程：

```python
# ✅ 路由只写正常逻辑，异常自动被拦截
@router.get("/detail")
async def get_news_detail(...):
    news_detail = await crud_get_news_detail(db, news_id)
    if not news_detail:
        raise HTTPException(404, "新闻不存在")   # ← 自动被处理器捕获
    ...
    # IntegrityError、连接超时等 → 自动返回统一格式的错误
```

## 二、四个处理器

### 文件结构

```
utils/
├── exception.py            ← 定义各个处理器函数
├── exception_handlers.py   ← 注册函数（把处理器绑定到 app）
```

### exception.py

```python
import traceback
from fastapi import HTTPException, Request
from fastapi.responses import JSONResponse
from sqlalchemy.exc import IntegrityError, SQLAlchemyError
from starlette import status

DEBUG_MODE = True  # 教学项目打开，生产环境改为 False


async def http_exception_handler(request: Request, exc: HTTPException):
    """处理业务异常（手动 raise 的 HTTPException）"""
    return JSONResponse(
        status_code=exc.status_code,
        content={
            "code": exc.status_code,
            "message": exc.detail,
            "data": None,
        },
    )


async def integrity_error_handler(request: Request, exc: IntegrityError):
    """处理唯一键冲突、外键约束等"""
    error_msg = str(exc.orig)
    if "Duplicate entry" in error_msg:
        detail = "用户名已存在"
    elif "FOREIGN KEY" in error_msg:
        detail = "关联数据不存在"
    else:
        detail = "数据约束冲突，请检查输入"

    error_data = None
    if DEBUG_MODE:
        error_data = {
            "error_type": "IntegrityError",
            "error_detail": error_msg,
            "path": str(request.url),
        }

    return JSONResponse(
        status_code=400,
        content={"code": 400, "message": detail, "data": error_data},
    )


async def sqlalchemy_error_handler(request: Request, exc: SQLAlchemyError):
    """处理数据库其他错误"""
    error_data = None
    if DEBUG_MODE:
        error_data = {
            "error_type": type(exc).__name__,
            "error_detail": str(exc),
            "traceback": traceback.format_exc(),
            "path": str(request.url),
        }

    return JSONResponse(
        status_code=500,
        content={"code": 500, "message": "数据库操作失败，请稍后重试", "data": error_data},
    )


async def general_exception_handler(request: Request, exc: Exception):
    """兜底：处理所有未捕获的异常"""
    error_data = None
    if DEBUG_MODE:
        error_data = {
            "error_type": type(exc).__name__,
            "error_detail": str(exc),
            "traceback": traceback.format_exc(),
            "path": str(request.url),
        }

    return JSONResponse(
        status_code=500,
        content={"code": 500, "message": "服务器内部错误", "data": error_data},
    )
```

### exception_handlers.py

```python
from fastapi import HTTPException
from sqlalchemy.exc import IntegrityError, SQLAlchemyError
from utils.exception import (
    http_exception_handler,
    integrity_error_handler,
    sqlalchemy_error_handler,
    general_exception_handler,
)


def register_exception_handlers(app):
    """注册顺序：子类在前，父类在后"""
    app.add_exception_handler(HTTPException, http_exception_handler)
    app.add_exception_handler(IntegrityError, integrity_error_handler)
    app.add_exception_handler(SQLAlchemyError, sqlalchemy_error_handler)
    app.add_exception_handler(Exception, general_exception_handler)
```

## 三、在 main.py 注册

```python
from utils.exception_handlers import register_exception_handlers

app = FastAPI()
register_exception_handlers(app)   # ← 加这一行
```

## 四、四个处理器的分工

| 处理器 | 捕获什么 | 返回 status | 返回 message |
|---|---|---|---|
| `http_exception_handler` | `raise HTTPException(404, "新闻不存在")` | `exc.status_code` | `exc.detail` |
| `integrity_error_handler` | 用户名重复、外键约束 | `400` | 根据错误类型判断 |
| `sqlalchemy_error_handler` | 其他数据库错误（连不上、SQL 语法错等） | `500` | `"数据库操作失败，请稍后重试"` |
| `general_exception_handler` | 所有未捕获的未知异常（兜底） | `500` | `"服务器内部错误"` |

---

## 五、为什么顺序是"子类在前、父类在后"

异常继承关系：

```
Exception
  └── SQLAlchemyError
        └── IntegrityError
  └── HTTPException
```

FastAPI 按注册顺序匹配，**先匹配先处理**。

```python
# ❌ 错误：Exception 在最前面
app.add_exception_handler(Exception, general)       # 所有异常都被它吃了
app.add_exception_handler(IntegrityError, integrity) # 永远不会执行到

# ✅ 正确：子类在前，父类在后
app.add_exception_handler(HTTPException, http)      # 具体业务异常
app.add_exception_handler(IntegrityError, integrity)  # 具体数据库约束
app.add_exception_handler(SQLAlchemyError, sqlalchemy) # 数据库大类
app.add_exception_handler(Exception, general)         # 兜底
```

## 六、DEBUG_MODE

```python
DEBUG_MODE = True   # 返回详细错误信息，方便调试
# 生产环境改为 False，data 为 None，不暴露内部细节
```

| 模式 | data 字段 |
|---|---|
| `True` | `{"error_type", "error_detail", "traceback", "path"}` |
| `False` | `None` |

## 七、和 success_response 统一

```python
# 成功
success_response(data=...)  # → {"code": 200, "message": "success", "data": ...}

# 异常 4xx（业务错误）
raise HTTPException(400, "参数错误")
# → {"code": 400, "message": "参数错误", "data": None}

# 异常 5xx（服务器错误）
# → {"code": 500, "message": "服务器内部错误", "data": ...}
```

前端只看 `code`：`200` 成功，其他失败。格式完全统一。

## 八、路由现在可以怎么写

```python
# 不需要 try-catch，直接 raise
raise HTTPException(status_code=404, detail="新闻不存在")

# 或者不处理，等全局兜底
# 比如 IntegrityError（用户名重复），自动返回 400
```
