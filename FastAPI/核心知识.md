## 1. 环境准备与第一个接口

### 安装依赖

```bash
pip install fastapi uvicorn
```

| 依赖 | 作用 |
| :--- | :--- |
| `fastapi` | Web 框架本体，负责路由、请求处理、数据验证、文档生成 |
| `uvicorn` | ASGI 服务器，用来运行 FastAPI 应用 |

### 最小示例

创建 `main.py`：

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/")
def read_root():
    return {"Hello": "World"}
```

### 启动项目

```bash
uvicorn main:app --reload
```

| 命令片段 | 含义 |
| :--- | :--- |
| `main` | Python 文件名，即 `main.py` |
| `app` | 文件中的 `FastAPI()` 实例变量名 |
| `--reload` | 开发模式热更新，代码变更后自动重启 |

访问 `http://127.0.0.1:8000`，返回：

```json
{"Hello": "World"}
```

## 2. 路由与请求参数

FastAPI 常见参数分为两类：

| 参数类型 | 位置 | 示例 |
| :--- | :--- | :--- |
| 路径参数 | URL 路径中 | `/items/1` |
| 查询参数 | URL 的 `?` 后面 | `/users/5?q=python` |

### 路径参数

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}
```

`item_id: int` 会触发 FastAPI 的自动类型校验。如果访问 `/items/abc`，FastAPI 会自动返回参数类型错误。

### 查询参数

```python
from fastapi import FastAPI

app = FastAPI()


@app.get("/users/{user_id}")
def read_user(user_id: int, q: str | None = None):
    if q:
        return {"user_id": user_id, "search_query": q}
    return {"user_id": user_id}
```

访问示例：

```text
http://127.0.0.1:8000/users/10?q=hello
```

### 自动交互文档

FastAPI 会自动生成 Swagger UI：

```text
http://127.0.0.1:8000/docs
```

在 `/docs` 页面可以直接点击接口，使用 `Try it out` 在线发送请求。

## 3. Pydantic 数据验证

Pydantic 用于定义请求体的数据结构，并自动完成校验、类型转换和错误响应。

### 为什么需要 Pydantic

它可以自动处理以下问题：

- 字段是否存在
- 类型是否正确
- 缺少必填字段时自动返回 `422`
- 自动生成接口文档中的请求体结构

### 请求体模型

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()


class Item(BaseModel):
    name: str
    description: str | None = None
    price: float
    tax: float | None = None


@app.post("/items/")
def create_item(item: Item):
    item_dict = item.model_dump()
    if item.tax:
        item_dict["total_price"] = item.price + item.tax
    return {"message": "商品创建成功", "item": item_dict}
```

### 请求示例

```json
{
  "name": "MacBook",
  "price": 9999.9,
  "description": "笔记本电脑"
}
```

### 关键点

| 写法 | 作用 |
| :--- | :--- |
| `BaseModel` | Pydantic 数据模型基类 |
| `name: str` | 必填字符串字段 |
| `description: str \| None = None` | 可选字符串字段 |
| `item.model_dump()` | 将 Pydantic 对象转换为字典，Pydantic v2 推荐写法 |

如果 `price` 传入字符串且无法转换为数字，FastAPI 会在进入业务函数前自动拦截。

## 4. 请求体进阶与 Field 注解

`Field` 用于给 Pydantic 模型字段增加约束和文档说明。它不只是校验工具，也会影响 `/docs` 中的字段展示。

常见使用场景：

- 限制字符串长度，例如用户名不能太短。
- 限制数字范围，例如价格不能为负数。
- 给 Swagger 文档补充字段中文说明。
- 给字段提供默认值、示例值。

### Field 基础示例

```python
from fastapi import Body, FastAPI
from pydantic import BaseModel, Field

app = FastAPI()


class Item(BaseModel):
    title: str = Field(..., min_length=2, max_length=20, description="商品的标题")
    description: str | None = Field(None, max_length=100, description="商品的详细描述")
    price: float = Field(..., gt=0, description="价格必须大于 0")
    tax: float | None = Field(default=0.1, examples=[0.05])


@app.post("/items/create")
async def create_item(item: Item):
    return {"item": item}
```

### Field 常用参数

| 参数            | 作用           | 示例                                 |
| :------------ | :----------- | :--------------------------------- |
| `...`         | 表示字段必填       | `Field(...)`                       |
| `default`     | 设置默认值        | `Field(default=0.1)`               |
| `min_length`  | 字符串最小长度      | `Field(..., min_length=2)`         |
| `max_length`  | 字符串最大长度      | `Field(..., max_length=20)`        |
| `gt`          | 大于           | `Field(..., gt=0)`                 |
| `ge`          | 大于等于         | `Field(..., ge=0)`                 |
| `lt`          | 小于           | `Field(..., lt=100)`               |
| `le`          | 小于等于         | `Field(..., le=100)`               |
| `description` | Swagger 字段说明 | `Field(..., description="商品标题")`   |
| `examples`    | Swagger 示例值  | `Field(..., examples=["MacBook"])` |

### Body 简单字段

如果请求体只有一个简单字段，不想专门定义 Pydantic 类，可以用 `Body(..., embed=True)`。

#### 有 `embed=True` 和没有的区别

```python
from fastapi import Body, FastAPI

app = FastAPI()


# 情况 A：没有 embed=True → 前端必须传纯值
@app.post("/items/plain")
async def get_item_plain(item_id: int = Body(...)):
    return {"item_id": item_id}


# 情况 B：有 embed=True → 前端必须包一层 JSON 对象
@app.post("/items/embedded")
async def get_item_embedded(item_id: int = Body(..., embed=True)):
    return {"item_id": item_id}
```

请求体对比：

| 写法                      | 前端发送的 JSON       | 说明            |
| :---------------------- | :--------------- | :------------ |
| `Body(...)`             | `1`              | 纯值，不包字段名      |
| `Body(..., embed=True)` | `{"item_id": 1}` | 包一层对象，键名就是参数名 |

> `embed=True` 的意思是：把参数名作为 key，把值包在 `{}` 里。不写的话 FastAPI 期望直接传裸值。

### 关键点

| 写法 | 含义 |
| :--- | :--- |
| `Field(...)` | 必填字段 |
| `Field(None)` | 可选字段，默认值为 `None` |
| `Field(default=0.1)` | 可选字段，默认值为 `0.1` |
| `gt=0` | 必须大于 0，不包含 0 |
| `ge=0` | 必须大于等于 0，包含 0 |
| `description="..."` | 展示在 Swagger 文档中，方便前后端协作 |

## 5. 响应类型

FastAPI 默认会把字典、列表等 Python 对象转换为 JSON。实际项目中还会返回 HTML、文件、图片、Excel、PDF 等不同类型的响应。

响应类型通常从 `fastapi.responses` 导入。

### JSONResponse

当需要手动设置状态码、响应头或响应内容时，可以使用 `JSONResponse`。

```python
from fastapi import FastAPI
from fastapi.responses import JSONResponse

app = FastAPI()


@app.get("/resp/json")
async def get_json():
    return JSONResponse(
        status_code=201,
        content={"message": "创建成功"},
        headers={"X-Custom-Header": "my-value"},
    )
```

### HTMLResponse

当接口返回 HTML 字符串时，建议在路由装饰器中声明 `response_class=HTMLResponse`，这样 Swagger 文档会更准确。

```python
from fastapi import FastAPI
from fastapi.responses import HTMLResponse

app = FastAPI()


@app.get("/resp/html", response_class=HTMLResponse)
async def get_html():
    return """
    <html>
        <head><title>我的网页</title></head>
        <body>
            <h1>你好！这是 FastAPI 返回的 HTML</h1>
        </body>
    </html>
    """
```

### FileResponse

返回图片、PDF、Excel 等文件时使用 `FileResponse`。

```python
from fastapi import FastAPI
from fastapi.responses import FileResponse

app = FastAPI()


@app.get("/resp/file")
async def get_file():
    path = "logo.png"
    return FileResponse(path, filename="下载后的图片.png")
```

### 响应类型对比

| 响应类型 | 适用场景 |
| :--- | :--- |
| 默认字典返回 | 普通 JSON API |
| `JSONResponse` | 自定义状态码、响应头、JSON 内容 |
| `HTMLResponse` | 返回网页 HTML |
| `FileResponse` | 文件下载或预览，如图片、PDF、Excel |

`FileResponse` 会以流式方式发送文件，不会一次性把大文件全部读入内存。

## 6. 响应模型与异常处理

响应模型和异常处理决定了接口是否安全、统一、方便前端调用。

### response_model 数据过滤

数据库模型可能包含敏感字段，例如 `password`。`response_model` 可以控制最终返回给前端的字段。

```python
from fastapi import FastAPI
from pydantic import BaseModel, EmailStr

app = FastAPI()


class UserOut(BaseModel):
    username: str
    email: EmailStr | None = None


@app.get("/users/me", response_model=UserOut)
async def get_current_user():
    user_in_db = {
        "username": "zhangsan",
        "email": "zs@example.com",
        "password": "secret_password_123",
    }
    return user_in_db
```

虽然 `return` 的数据中有 `password`，但最终响应会被 `UserOut` 过滤，前端只能看到 `username` 和 `email`。

### HTTPException

业务逻辑中主动返回错误时，使用 `HTTPException`。

```python
from fastapi import FastAPI, HTTPException

app = FastAPI()


@app.get("/items/{item_id}")
async def read_item(item_id: int):
    if item_id != 1:
        raise HTTPException(status_code=404, detail="宝贝不存在，换个 ID 试试吧")
    return {"item": "找到宝贝了"}
```

`raise HTTPException(...)` 会立即停止当前函数，并返回指定状态码和错误信息。

### 自定义全局异常处理器

定义自定义异常处理器需要 **3 步**，下面用"库存不足"这个业务场景一步步演示。

#### 第 1 步：定义自定义异常类

先写一个继承 `Exception` 的类，用来表示"库存不足"这种业务错误。`__init__` 的参数就是你想在错误信息里携带的数据：

```python
class InsufficientStockError(Exception):
    def __init__(self, product_id: int, current_stock: int):
        self.product_id = product_id
        self.current_stock = current_stock
```

> **为什么要继承 `Exception`？**
> 
> 因为 Python 的 `raise` 只能抛出 `Exception` 的子类。如果你不继承它：
> 
> ```python
> class Foo:
>     pass
> 
> raise Foo()
> # TypeError: exceptions must derive from BaseException
> ```
> 
> 连 `raise` 本身都会报错，根本到不了 FastAPI 那一层。
> 
> 另外 `@app.exception_handler(XxxError)` 的工作原理是：FastAPI 捕获到异常后，用 `isinstance(exc, XxxError)` 判断是否匹配你注册的类型。只有继承自 `Exception` 的对象才能走通这套异常捕获机制。
> 
> 所以这个类虽然看起来"不做任何事"，但它必须继承 `Exception`，否则无法被 `raise` 抛出，也无法被 FastAPI 捕获。

#### 第 2 步：用 `@app.exception_handler()` 注册处理器

写一个函数，用 `@app.exception_handler(InsufficientStockError)` 装饰它。当程序抛出 `InsufficientStockError` 时，FastAPI 会自动调用这个函数来处理：

```python
from fastapi import Request
from fastapi.responses import JSONResponse


@app.exception_handler(InsufficientStockError)
async def insufficient_stock_handler(request: Request, exc: InsufficientStockError):
    return JSONResponse(
        status_code=400,
        content={
            "code": "INSUFFICIENT_STOCK",
            "message": "商品库存不足",
            "data": {
                "product_id": exc.product_id,
                "current_stock": exc.current_stock,
            },
        },
    )
```

> 函数有两个参数：
> - `request`：发生错误的请求信息（一般不用动它）。
> - `exc`：你抛出来的异常实例，可以通过 `exc.product_id` 拿到第 1 步里存的数据。

#### 第 3 步：在业务代码中 `raise` 触发

在接口函数里，当库存不够时直接 `raise` 这个异常。程序会立即停下，跳到第 2 步的处理器去：

```python
@app.post("/orders/")
async def create_order(product_id: int, quantity: int):
    current_stock = 3  # 模拟数据库查询结果

    if quantity > current_stock:
        raise InsufficientStockError(
            product_id=product_id,
            current_stock=current_stock,
        )

    return {"message": "下单成功"}
```

#### 请求与响应示例

请求：

```text
POST /orders/?product_id=1001&quantity=5
```

响应（前端收到的统一格式）：

```json
{
  "code": "INSUFFICIENT_STOCK",
  "message": "商品库存不足",
  "data": {
    "product_id": 1001,
    "current_stock": 3
  }
}
```

#### 三步总结

| 步骤 | 做什么 | 关键代码 |
| :--- | :--- | :--- |
| 1 | 定义异常类，存错误数据 | `class XxxError(Exception):` |
| 2 | 注册处理器，返回统一格式的 JSON | `@app.exception_handler(XxxError)` |
| 3 | 在业务中 `raise` 触发 | `raise XxxError(...)` |

```json
{
  "code": "INSUFFICIENT_STOCK",
  "message": "商品库存不足",
  "data": {
    "product_id": 1001,
    "current_stock": 3
  }
}
```

### 关键点

| 写法                            | 作用                                    |
| :---------------------------- | :------------------------------------ |
| `response_model=UserOut`      | 控制响应字段，过滤敏感数据                         |
| `HTTPException`               | 主动抛出 HTTP 错误                          |
| `raise`                       | 抛出异常并停止当前函数继续执行                       |
| `@app.exception_handler(...)` | 注册全局异常处理器，统一处理某类业务异常                  |
| 统一错误格式                        | 方便前端固定按 `{code, message, data}` 等结构处理 |

## 7. 异步编程 async/await

FastAPI 基于 ASGI，天然支持异步处理。异步适合处理网络请求、数据库查询、文件 IO 等等待型任务。

### 核心理解

| 模式 | 行为 |
| :--- | :--- |
| 同步 | 当前任务没结束前，线程一直等待 |
| 异步 | 当前任务等待 IO 时，把执行权让给其他请求 |

### 异步路由示例

```python
import asyncio
from fastapi import FastAPI

app = FastAPI()


@app.get("/async-task")
async def perform_async_task():
    print("任务开始...")
    await asyncio.sleep(2)
    print("任务结束！")
    return {"status": "success", "message": "任务处理完成"}
```

### 注意事项

- `async def` 声明异步函数。
- `await` 只能等待可 await 的异步操作。
- 不要在 `async def` 中使用 `time.sleep()` 这类阻塞代码。
- 同步路由 `def` 也能运行，FastAPI 会放入线程池处理。

## 8. 依赖注入 Depends

依赖注入用于把公共逻辑提取成独立函数，并在接口执行前自动准备好。

常见用途：

- 获取数据库会话
- 解析分页参数
- 校验登录用户
- 权限检查
- 复用公共查询条件

### 基础示例

```python
from fastapi import Depends, FastAPI

app = FastAPI()


def common_parameters(skip: int = 0, limit: int = 100):
    return {"skip": skip, "limit": limit}


@app.get("/items/")
def read_items(commons: dict = Depends(common_parameters)):
    return {"message": "这是列表接口", "params": commons}


@app.get("/users/")
def read_users(commons: dict = Depends(common_parameters)):
    return {"message": "这是用户接口", "params": commons}
```

### Java 对比：相当于 Spring 的 `@Autowired`

如果你学过 Spring，`Depends` 和 Spring 的依赖注入是同一类东西，但更轻量：

| 对比项 | Spring (Java) | FastAPI Depends (Python) |
| :--- | :--- | :--- |
| 注入目标 | 类的字段 / 构造器参数 | 函数的参数 |
| 声明方式 | `@Autowired` 或构造器注入 | `Depends(某个函数)` |
| 依赖来源 | IoC 容器管理的 Bean | 普通函数的返回值 |
| 注入时机 | 启动时创建 Bean 就注入 | 每次请求进来时执行函数 |
| 生命周期管理 | 框架全权管理 | 你自己控制，用 `yield` 管理资源 |

**Spring 的 DI 是"找 Bean"**：启动时扫描所有类，发现 `@Autowired` 就去容器里找匹配的 Bean 塞进来。

**FastAPI 的 Depends 是"先执行这个函数"**：`Depends(common_parameters)` 的意思是——在进入 `list_users` 之前，先执行 `common_parameters(skip, limit)`，把返回值作为参数传进去。**本质就是函数嵌套调用，只是 FastAPI 帮你自动做了。**

### 关键点

| 写法 | 含义 |
| :--- | :--- |
| `Depends(common_parameters)` | 请求进入接口前，先执行依赖函数 |
| `commons: dict` | 接收依赖函数的返回值 |
| 公共依赖函数 | 修改一次，多处接口生效 |

## 9. SQLAlchemy 异步数据库基础

FastAPI 常用异步数据库组合：`SQLAlchemy 2.0+` + `aiomysql`。

更完整的 ORM 增删改查进阶内容见：`FastAPI/SQLAlchemy.md`。

### 安装依赖

```bash
pip install sqlalchemy aiomysql
```

### 数据库配置 `database.py`

```python
from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine
from sqlalchemy.orm import DeclarativeBase

SQLALCHEMY_DATABASE_URL = "mysql+aiomysql://root:123456@127.0.0.1:3306/test_db"

engine = create_async_engine(
    SQLALCHEMY_DATABASE_URL,
    echo=True,
)

AsyncSessionLocal = async_sessionmaker(
    bind=engine,
    class_=AsyncSession,
    expire_on_commit=False,
)


class Base(DeclarativeBase):
    pass
```

### 定义模型

```python
from sqlalchemy import String
from sqlalchemy.orm import Mapped, mapped_column


class User(Base):
    __tablename__ = "users"

    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False)
    email: Mapped[str | None] = mapped_column(String(100), nullable=True)
```

### 初始化数据库表

```python
async def init_db():
    async with engine.begin() as conn:
        await conn.run_sync(Base.metadata.create_all)
```

### 数据库会话依赖

```python
async def get_db():
    async with AsyncSessionLocal() as session:
        try:
            yield session
        finally:
            await session.close()
```

`yield session` 的作用是把数据库会话交给接口使用。接口结束后，代码回到 `yield` 后面继续执行清理逻辑。

### 创建用户接口

```python
from fastapi import Depends, FastAPI
from pydantic import BaseModel
from sqlalchemy.ext.asyncio import AsyncSession

from database import User, get_db, init_db

app = FastAPI()


class UserCreate(BaseModel):
    username: str
    email: str | None = None


@app.on_event("startup")
async def on_startup():
    await init_db()


@app.post("/users/")
async def create_user(user_data: UserCreate, db: AsyncSession = Depends(get_db)):
    new_user = User(username=user_data.username, email=user_data.email)
    db.add(new_user)
    await db.commit()
    await db.refresh(new_user)
    return {"id": new_user.id, "username": new_user.username}
```

### 查询用户接口

```python
from sqlalchemy import select


@app.get("/users/")
async def read_users(db: AsyncSession = Depends(get_db)):
    stmt = select(User)
    result = await db.execute(stmt)
    return result.scalars().all()


@app.get("/users/{user_id}")
async def read_user(user_id: int, db: AsyncSession = Depends(get_db)):
    stmt = select(User).where(User.id == user_id)
    result = await db.execute(stmt)
    user = result.scalars().first()

    if user is None:
        return {"message": "用户不存在"}

    return user
```

### SQLAlchemy 基础概念

| 概念                      | 说明                  |
| :---------------------- | :------------------ |
| `mysql+aiomysql`        | MySQL 异步连接协议        |
| `create_async_engine()` | 创建异步数据库引擎           |
| `AsyncSession`          | 异步数据库会话             |
| `DeclarativeBase`       | SQLAlchemy 2.0 模型基类 |
| `Mapped`                | 字段的 Python 类型声明     |
| `mapped_column()`       | 字段的数据库列配置           |
| `select(User)`          | 构造查询语句              |
| `db.execute(stmt)`      | 执行 SQL 语句           |
| `scalars().all()`       | 提取模型对象列表            |
| `scalars().first()`     | 提取第一个模型对象           |

## 10. 项目拆分 APIRouter

当接口数量变多时，不应把所有代码都放在 `main.py` 中。推荐按职责拆分模块。

### 推荐目录结构

```text
my_project/
├── main.py
├── database.py
├── models.py
├── schemas.py
└── routers/
    └── user.py
```

| 文件 | 职责 |
| :--- | :--- |
| `main.py` | 应用入口，注册路由 |
| `database.py` | 数据库引擎、会话、初始化逻辑 |
| `models.py` | SQLAlchemy 数据库模型 |
| `schemas.py` | Pydantic 请求/响应模型 |
| `routers/user.py` | 用户模块接口 |

### 子路由 `routers/user.py`

```python
from fastapi import APIRouter, Depends
from sqlalchemy.ext.asyncio import AsyncSession

from database import get_db

router = APIRouter(
    prefix="/users",
    tags=["用户管理"],
)


@router.get("/")
async def list_users(db: AsyncSession = Depends(get_db)):
    return {"message": "这是用户列表"}


@router.post("/")
async def add_user():
    return {"message": "添加用户"}
```

### 主入口 `main.py`

```python
from fastapi import FastAPI
from routers import user

app = FastAPI()

app.include_router(user.router)


@app.get("/")
def index():
    return {"msg": "欢迎来到主页"}
```

### 关键点

| 写法 | 作用 |
| :--- | :--- |
| `APIRouter()` | 创建模块级路由对象 |
| `prefix="/users"` | 给一组接口统一增加路径前缀 |
| `tags=["用户管理"]` | 在 Swagger 文档中分组 |
| `app.include_router()` | 把子路由注册到主应用 |

## 11. AI 模型部署接口

FastAPI 常用于把 AI 模型封装成 HTTP API。

核心原则：不要在接口函数中重复加载模型。模型应在应用启动或模块导入时加载一次，接口收到请求后直接调用内存中的模型。

### 模拟模型 `ai_model.py`

```python
import asyncio


class SentimentModel:
    def __init__(self):
        self.model_name = "My_AI_Model_v1"

    async def predict(self, text: str):
        await asyncio.sleep(0.5)
        if "好" in text or "赞" in text:
            return "Positive (积极)"
        return "Negative (消极)"


model = SentimentModel()
```

### 封装预测接口

```python
from fastapi import Body, FastAPI

from ai_model import model

app = FastAPI()


@app.post("/ai/predict")
async def get_prediction(text: str = Body(..., embed=True)):
    result = await model.predict(text)
    return {
        "model": model.model_name,
        "input": text,
        "prediction": result,
    }
```

### 请求体示例

```json
{"text": "这个课程真的太赞了！"}
```

### 关键点

| 概念 | 说明 |
| :--- | :--- |
| 模型单例 | 模型只加载一次，所有请求复用同一个实例 |
| `Body(..., embed=True)` | 从 JSON 中读取指定字段（参数名作为 key，如 `{"text": "..."}`） |
| CPU 密集型推理 | 可使用同步路由、线程池或任务队列避免阻塞事件循环 |

## 12. 生产环境部署

开发环境常用：

```bash
uvicorn main:app --reload
```

生产环境不建议使用 `--reload`，也不建议只跑单进程。

### Gunicorn + Uvicorn Worker

安装：

```bash
pip install gunicorn
```

Linux 服务器启动命令：

```bash
gunicorn main:app -w 4 -k uvicorn.workers.UvicornWorker --bind 0.0.0.0:8000
```

| 参数 | 含义 |
| :--- | :--- |
| `-w 4` | 启动 4 个工作进程 |
| `-k uvicorn.workers.UvicornWorker` | 使用 Uvicorn 异步 worker |
| `--bind 0.0.0.0:8000` | 监听所有网卡的 8000 端口 |

### Dockerfile 示例

```dockerfile
FROM python:3.10-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["gunicorn", "main:app", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "--bind", "0.0.0.0:8000"]
```

### 构建与运行

```bash
docker build -t my_fastapi_app .
docker run -d -p 8000:8000 my_fastapi_app
```

### 部署概念

| 工具       | 作用                    |
| :------- | :-------------------- |
| Uvicorn  | ASGI 服务器，负责运行 FastAPI |
| Gunicorn | 进程管理器，负责多 worker 管理   |
| Docker   | 打包代码、依赖和运行环境，降低部署环境差异 |

```python
from fastapi.testclient import TestClient

from main import app

client = TestClient(app)

    assert response.json() == {"Hello": "World"}


def test_create_user():
    response = client.post(
        "/users/",

    )
    assert response.status_code == 200
    assert "id" in response.json()["user"]
```

运行测试：

```bash
pytest
```

### 后台任务 BackgroundTasks

适合发送邮件、写日志、处理图片等不需要让用户等待完成的任务。

```python
from fastapi import BackgroundTasks, FastAPI

app = FastAPI()


def write_log(message: str):
    with open("log.txt", mode="a", encoding="utf-8") as log:
        log.write(message + "\n")


@app.post("/send-notification/{email}")
async def send_notification(email: str, background_tasks: BackgroundTasks):
    background_tasks.add_task(write_log, f"通知发送给: {email}")
    return {"message": "通知已发出，后台正在处理"}
```

### 中间件 Middleware

中间件可以在请求进入接口前和响应返回前执行统一逻辑，例如统计耗时。

```python
import time

from fastapi import Request


@app.middleware("http")
async def add_process_time_header(request: Request, call_next):
    start_time = time.time()
    response = await call_next(request)
    process_time = time.time() - start_time
    response.headers["X-Process-Time"] = str(process_time)
    print(f"请求路径: {request.url.path} 耗时: {process_time:.4f}s")
    return response
```

### 优化方向

| 能力 | 用途 |
| :--- | :--- |
| 自动化测试 | 防止改动破坏旧功能 |
| 后台任务 | 将非核心耗时任务延后执行 |
| 中间件 | 统一处理日志、耗时、鉴权、跨域等逻辑 |

### FastAPI 核心写法

| 写法 | 作用 |
