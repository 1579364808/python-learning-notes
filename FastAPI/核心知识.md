### Step 2: 路由与请求参数

FastAPI 处理请求参数主要分为两类：**路径参数 (Path Parameters)** 和 **查询参数 (Query Parameters)**。

#### 1. 路径参数 (Path Parameters)
直接在 URL 路径中声明变量。

#### 2. 查询参数 (Query Parameters)
当函数参数不在路径中时，FastAPI 会自动将其识别为查询参数（即 URL 中 `?` 之后的部分）。

---

### 动手实践

请修改你的 `main.py` 文件，替换为以下代码：

```python
from fastapi import FastAPI
from typing import Union

app = FastAPI()

# 1. 路径参数：通过 {item_id} 声明，类型提示为 int，FastAPI 会自动校验
@app.get("/items/{item_id}")
def read_item(item_id: int):
    return {"item_id": item_id}

# 2. 混合使用：路径参数 + 查询参数
# 如果 URL 是 /users/5?q=python，那么 item_id=5, q="python"
# q 设置了默认值 None，代表它是可选的
@app.get("/users/{user_id}")
def read_user(user_id: int, q: Union[str, None] = None):
    if q:
        return {"user_id": user_id, "search_query": q}
    return {"user_id": user_id}
```

### 关键点解析：
1.  **自动类型转换**：注意 `item_id: int`。如果你在浏览器输入 `http://127.0.0.1:8000/items/abc`，FastAPI 会自动返回错误提示，告诉你 `item_id` 必须是整数，无需手动解析类型！
2.  **可选参数**：通过 `q: Union[str, None] = None`（或者在 Python 3.10+ 中直接写 `q: str | None = None`），我们定义了一个可选查询参数。如果用户不传 `q`，它的值就是 `None`。

### Step 3: Pydantic 数据验证

#### 为什么需要 Pydantic？
如果不使用 Pydantic，你需要手动编写代码去检查：
*   请求里有没有 `name` 字段？
*   `price` 是不是数字？如果用户传了字符串怎么办？
*   如果缺少必填字段，是不是要手动返回 422 错误？

Pydantic 帮你自动完成了这一切。你只需要定义一个**模型（Model）**，它就会自动校验、转换数据，甚至自动生成文档。

---

### 动手实践

请将你的 `main.py` 代码修改如下：

```python
from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI()

# 1. 定义数据模型
# 继承 BaseModel，这样这个类就拥有了自动验证数据的功能
class Item(BaseModel):
    name: str           # 必填字段
    description: str = None  # 可选字段，默认 None
    price: float        # 必填，且必须是数字
    tax: float = None   # 可选

# 2. 定义 POST 接口
@app.post("/items/")
def create_item(item: Item):
    # FastAPI 自动将请求的 JSON 解析成了 item 对象
    # 你可以直接访问属性，比如 item.name
    item_dict = item.dict() # 转换为字典
    if item.tax:
        item_dict.update({"total_price": item.price + item.tax})
    return {"message": "商品创建成功", "item": item_dict}
```

### 关键点解析：
1.  **`BaseModel`**：这是 Pydantic 的基类。我们通过定义类型（`str`, `float`）告诉 FastAPI 数据的结构。
2.  **`@app.post`**：处理 POST 请求，通常用于提交数据。
3.  **自动解析**：在 `create_item(item: Item)` 函数中，FastAPI 会自动把用户发来的 JSON 转化为 `Item` 类的实例。如果用户发送的 JSON 格式不对（比如 `price` 传了 "abc"），FastAPI 会**自动拦截并返回详细的错误信息**，你的函数逻辑代码甚至根本不会运行，非常安全！

欢迎来到 **Step 4：异步编程 (Async/Await)**。

这是 FastAPI 之所以被称为“高性能框架”的关键原因。FastAPI 是完全基于 `asyncio` 构建的，它允许你处理高并发请求，而不必为每个请求开启一个庞大的线程，极大地节省了系统资源。

### 为什么需要异步？

想象你在一家咖啡店：
*   **同步编程 (普通代码)**：你点了一杯咖啡，站在前台死等，直到咖啡做好才离开。这期间，哪怕后面排了长队，收银员也无法为下一个人点单（**阻塞**）。
*   **异步编程 (async/await)**：你点了一杯咖啡，服务员给你一个“呼叫器”（Promise/Future），你就可以去旁边坐下玩手机了。服务员立刻可以为下一个人点单。当咖啡做好时，呼叫器响了，你再去取咖啡（**非阻塞**）。

### 动手实践

请将 `main.py` 修改为以下代码，我们模拟一个耗时的 IO 操作（比如数据库查询或请求外部 API）：

```python
import asyncio
from fastapi import FastAPI

app = FastAPI()

# 使用 async def 定义路由
@app.get("/async-task")
async def perform_async_task():
    print("任务开始...")
    # await 关键字用于挂起当前函数，等待耗时任务完成
    # 在这期间，FastAPI 可以去处理其他进来的请求
    await asyncio.sleep(2)  # 模拟耗时 2 秒的 IO 操作
    print("任务结束！")
    return {"status": "success", "message": "任务处理完成"}
```

### 核心知识点：

1.  **`async def`**：声明这是一个异步路由处理函数。
2.  **`await`**：这是最关键的。当代码执行到 `await asyncio.sleep(2)` 时，程序会告诉 FastAPI：“我现在要等这个任务完成，你可以先去处理别的事情（其他用户请求）”。
3.  **不能阻塞 Event Loop**：在 `async def` 函数中，**严禁**使用同步阻塞代码（比如 `time.sleep(2)`），因为这会卡死整个服务器，让所有请求都停下来等你，直接抵消了异步的优势。

4.  **进阶思考**：如果你在代码中写 `def` 而不是 `async def`，FastAPI 依然可以运行（它会把同步函数放在单独的线程池里运行），但如果你能使用 `async def`，它的执行效率会更高。

欢迎来到 **Step 5：依赖注入 (Dependency Injection)**。

这是 FastAPI 中最强大、也最容易让人感到“哇，原来可以这样写”的设计模式。

### 什么是依赖注入 (DI)？

简单来说，**依赖注入就是“把你需要用到的工具（依赖），在函数执行前先准备好，并自动传给函数”**。

在 Web 开发中，我们经常会遇到这种情况：多个接口都需要同一个功能（例如：检查用户是否登录、获取数据库连接、解析分页参数）。如果你在每个函数里都重复写一遍代码，代码会变得冗余且难以维护。

**依赖注入**让你把这些逻辑抽象成一个独立的函数，然后用 `Depends` 来“注入”它。

---

### 动手实践

假设我们有多个接口都需要分页参数（`skip` 和 `limit`），我们把它提取出来。

请修改 `main.py`：

```python
from fastapi import FastAPI, Depends

app = FastAPI()

# 1. 定义一个依赖函数
# 这个函数的作用就是提供分页参数，它可以做任何逻辑（比如验证权限、连接数据库）
def common_parameters(skip: int = 0, limit: int = 100):
    return {"skip": skip, "limit": limit}

# 2. 在路由中使用 Depends 注入依赖
@app.get("/items/")
def read_items(commons: dict = Depends(common_parameters)):
    return {"message": "这是列表接口", "params": commons}

@app.get("/users/")
def read_users(commons: dict = Depends(common_parameters)):
    return {"message": "这是用户接口", "params": commons}
```

### 关键点解析：

1.  **`Depends(common_parameters)`**：这就是核心。你不需要在 `read_items` 函数里手动去解析 `skip` 和 `limit`，FastAPI 会在调用 `read_items` 之前，先执行 `common_parameters`，并将结果传递给 `commons` 参数。
2.  **代码复用**：如果以后要改分页规则（比如默认 limit 改为 50），你只需要改 `common_parameters` 一个地方，所有相关的接口都会自动更新。
3.  **可测试性**：你可以轻松地为 `common_parameters` 写单元测试，而不需要启动整个 Web 服务。

#### 2. 编写数据库配置

请看代码实现：

```python
from sqlalchemy.ext.asyncio import create_async_engine, AsyncSession, async_sessionmaker
from sqlalchemy.orm import DeclarativeBase

# 1. 数据库连接地址 (注意：协议是 mysql+aiomysql)
# 格式：mysql+aiomysql://用户名:密码@地址:端口/数据库名
SQLALCHEMY_DATABASE_URL = "mysql+aiomysql://root:123456@127.0.0.1:3306/test_db"

# 2. 创建异步引擎
engine = create_async_engine(
    SQLALCHEMY_DATABASE_URL,
    echo=True,  # 设置为 True 会在终端打印 SQL 语句，方便调试
)

# 3. 创建异步会话工厂
# expire_on_commit=False 是为了在 commit 后还能访问对象属性
AsyncSessionLocal = async_sessionmaker(
    bind=engine, 
    class_=AsyncSession, 
    expire_on_commit=False
)

# 4. 定义模型基类
class Base(DeclarativeBase):
    pass
```

### 关键点解析：
1.  **`mysql+aiomysql`**：这是异步连接的关键。普通的 `mysql+pymysql` 是同步的，会阻塞异步循环。
2.  **`create_async_engine`**：创建一个支持异步的引擎。
3.  **`DeclarativeBase`**：这是 SQLAlchemy 2.0 的新写法，所有数据库模型类都要继承它。

### Step 6.2：定义数据库模型（把 Python 类变成数据库表）

在 ORM 的世界里，**一个 Python 类 = 数据库里的一张表**。
类的**属性**（变量） = 表里的**字段**（列）。

我们要创建一个简单的“用户表” `users`，包含：ID、用户名、邮箱。

#### 1. 编写模型代码
请在刚才的 `database.py` 后面继续添加这段代码：

```python
from sqlalchemy import String
from sqlalchemy.orm import Mapped, mapped_column

# 我们定义的模型类，必须继承之前写好的 Base
class User(Base):
    __tablename__ = "users"  # 这就是数据库里真实的表名

    # id: 整数类型，主键，自动递增
    id: Mapped[int] = mapped_column(primary_key=True, autoincrement=True)
    
    # username: 字符串(长度50)，不能为空，值必须唯一
    username: Mapped[str] = mapped_column(String(50), unique=True, nullable=False)
    
    # email: 字符串(长度100)，可以为空
    email: Mapped[str] = mapped_column(String(100), nullable=True)
```

#### 2. 怎么让表在数据库里生成？
代码写好了，但 MySQL 里现在还没这张表。在 FastAPI 启动时，我们可以让它自动去创建。

在 `database.py` 中添加这个简单的工具函数：

```python
# 这个函数的作用是：根据上面定义的类，在数据库里把表建出来
async def init_db():
    async with engine.begin() as conn:
        # 这一行代码会自动去找继承了 Base 的所有类，并创建表
        await conn.run_sync(Base.metadata.create_all)
```

---

### 简单易懂的解释：
*   **`Mapped[str]`**：这是告诉 Python 和编辑器，这个字段存的是字符串。
*   **`mapped_column`**：这是给数据库看的“说明书”。比如 `String(50)` 告诉数据库分配 50 个字符的空间；`primary_key=True` 告诉数据库这是主键（身份证号）。
*   **`init_db`**：就像是一个“一键建表”的开关。
