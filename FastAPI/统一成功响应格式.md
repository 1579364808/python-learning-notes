# FastAPI 封装统一成功响应格式

## 一、为什么需要统一响应格式

没有统一封装之前，每个接口返回的格式五花八门：

```python
# 接口A 返回字典
return {"code": 200, "data": user_info}

# 接口B 返回 Pydantic 模型
return UserInfoResponse(id=1, username="foo")

# 接口C 返回 ORM 对象
return db_user
```

前端对接困难，不知道每个接口的结构是否一致。

**目标**：所有接口统一返回 `{ "code": 200, "message": "...", "data": ... }`。

---

## 二、定义统一外壳函数

```python
# utils/response.py

from fastapi.responses import JSONResponse
from fastapi.encoders import jsonable_encoder


def success_response(message: str = "success", data=None):
    content = {
        "code": 200,
        "message": message,
        "data": data,
    }
    return JSONResponse(content=jsonable_encoder(content))
```

---

## 三、问题：data 的类型不统一

调用时 data 可能是各种东西：

```python
# 普通字典
success_response(data={"id": 1, "name": "foo"})

# Pydantic 模型
success_response(data=UserInfoResponse(id=1, username="foo"))

# SQLAlchemy ORM 对象
success_response(data=db_user)

# datetime / UUID / Decimal 等
success_response(data=created_at)
```

JSON 的 `json.dumps()` 不认识这些类型，直接返回会报错：

```
TypeError: Object of type datetime is not JSON serializable
```

---

## 四、jsonable_encoder 解决序列化问题

`jsonable_encoder` 是 FastAPI 提供的工具，递归地将所有特殊类型转成 JSON 原生类型：

| 原始类型 | 转换后 |
|---|---|
| `Pydantic 模型` | `字典` |
| `datetime` | `"2026-05-25T12:00:00"` |
| `UUID` | `"550e8400-e29b-..."` |
| `Decimal("19.99")` | `19.99` |
| `set([1,2,3])` | `[1,2,3]` |
| `SQLAlchemy ORM 对象` | `对应字典` |
| 嵌套对象 | 递归全部展平 |

```python
encoded = jsonable_encoder(content)
# encoded 现在是纯字典，全是 str/int/float/list/dict/None

# JSONResponse 底层是 json.dumps(encoded) → 100% 不会报错
return JSONResponse(content=encoded)

# 一行搞定：
return JSONResponse(content=jsonable_encoder(content))
```

## 五、Pydantic model_validate：ORM 对象 → Pydantic 模型

### 问题

ORM 对象（SQLAlchemy 模型实例）不能直接传给 Pydantic：

```python
user = User(id=1, username="jack", nickname="杰克")

# ❌ 不行
UserInfoResponse(user)

# ✅ 关键字传参可以，但太麻烦
UserInfoResponse(id=user.id, username=user.username, nickname=user.nickname)
```

### 解决：from_attributes=True + model_validate

```python
from pydantic import BaseModel, ConfigDict

class UserInfoResponse(BaseModel):
    id: int
    username: str
    nickname: str | None = None

    # 开关：允许从对象的属性（点号）读值
    model_config = ConfigDict(from_attributes=True)

# 使用
user_info = UserInfoResponse.model_validate(user)
# Pydantic 内部做了：
#   id = user.id
#   username = user.username
#   nickname = user.nickname
```


---

## 六、ConfigDict 配置详解

`model_config = ConfigDict(...)` 是 Pydantic v2 的模型配置入口，控制整个模型的行为。

### 1. `from_attributes=True` （最常用）

**作用**：允许从对象的属性（`点号`）取数据，而不是只接受字典或关键字参数。

```python
# ❌ 没有 from_attributes
UserInfoResponse(user)                           # 报错！不知道 user 是什么
UserInfoResponse(id=user.id, username=user.username)  # 能行，但手动写

# ✅ 有 from_attributes
UserInfoResponse.model_validate(user)
# 自动：id = user.id, username = user.username, ...
```

**本质**：`model_validate(obj)` 等价于手动写关键字参数：

```python
# 手动（麻烦）
UserInfoResponse(id=user.id, username=user.username, nickname=user.nickname)

# 自动（一行）
UserInfoResponse.model_validate(user)
#       ↑ 遍历 schema 的所有字段，从 user 身上读同名字段
```

**适用场景**：路由里拿到的是 **ORM 对象**（SQLAlchemy），需要转成 Pydantic 再返回。

---

### 2. `populate_by_name=True` （配合 alias 使用）

**作用**：有 `alias` 别名的字段，既可以用别名赋值，也可以用 Python 原名赋值。

```python
# schema 定义
publish_time: datetime = Field(..., alias="publishTime")

# ❌ 没有 populate_by_name
NewsItemResponse(publish_time="...")   # 报错！只能写 publishTime
NewsItemResponse(publishTime="...")    # 能行，但代码里写驼峰很奇怪

# ✅ 有 populate_by_name
NewsItemResponse(publish_time="...")   # Python 风格，能行
NewsItemResponse(publishTime="...")    # 别名风格，也能行
```

**为什么需要**：两个地方用不同的名字：

| 场景 | 用哪个名字 | 示例 |
|---|---|---|
| `model_validate(obj)` 从 ORM 读属性 | **Python 名** | `obj.publish_time` |
| 手动构造模型（如 `UserAuthResponse(token=token, user_info=...)`） | **Python 名** | `UserAuthResponse(user_info=...)` |
| 序列化成 JSON 给前端 | **别名**（驼峰） | `{"publishTime": "..."}` |

**没有 `populate_by_name` 的问题**：

```python
# users.py
class UserAuthResponse(BaseModel):
    token: str
    user_info: UserInfoResponse = Field(..., alias="userInfo")

# router 里：
response_data = UserAuthResponse(token=token, user_info=user_info)
#                                          ↑ 用的是 Python 名 user_info
# 没有 populate_by_name → 报错，必须写成 userInfo
# 有 populate_by_name  → 正常，user_info 和 userInfo 都认
```

---

### 3. 两个配置的关系

```python
model_config = ConfigDict(
    populate_by_name=True,   # ① 构造时认 Python 名
    from_attributes=True,    # ② 允许从 ORM 对象读属性
)
```

- `from_attributes=True` — **必须和 `model_validate` 配合**，否则 `model_validate(obj)` 会报错
- `populate_by_name=True` — **必须和 `alias` 配合**，否则代码里只能用驼峰名构造
- 两者独立，可以只用其中一个

### 4. 何时用哪个

| 场景 | from_attributes | populate_by_name |
|---|---|---|
| ORM 对象 → Pydantic（最常用） | ✅ 要 | 建议加 |
| 纯字典/JSON → Pydantic | ❌ 不用 | 建议加 |
| 有 alias 的字段在代码里构造 | 按需 | ✅ 要 |
| 没有 alias 的字段 | 按需 | ❌ 不用 |

**本项目最佳实践**：所有响应模型都加两者，省心。

---

### 5. model_validate 时字段不匹配的规则

```python
user = User(id=1, username="jack", password="123456", phone="138xxx")

class UserOut(BaseModel):
    id: int
    username: str
    nickname: Optional[str] = None   # 有默认值
    # 没有 password, phone

out = UserOut.model_validate(user)
```

| 场景 | 结果 |
|---|---|
| ORM 属性多，schema 字段少（如 `password`、`phone`） | **自动忽略**，安全过滤 |
| ORM 属性少，schema 字段有默认值（如 `nickname`） | **用默认值填充**，不报错 |
| ORM 属性少，schema 字段无默认值（如 `id`） | **报 ValidationError**，字段缺失 |

所以可选字段建议都给默认值 `Optional[str] = None`，避免 ORM 查询漏列时崩掉。

## 七、整体调用链

```
路由函数：
    return success_response(data=response_data)

步骤分解：
    ① ORM 对象 → Pydantic 模型
       user_info = UserInfoResponse.model_validate(user)

    ② 组装成统一响应
       response_data = UserAuthResponse(token=token, user_info=user_info)

    ③ 传给 success_response
       content = {"code": 200, "message": "注册成功", "data": response_data}

    ④ jsonable_encoder 递归展平
       → {"code": 200, "message": "注册成功", "data": {"id": 1, "username": "jack", ...}}

    ⑤ JSONResponse 序列化 + 设置 Content-Type: application/json
       → 发送给客户端
```

## 八、补充：response_model（FastAPI 路由装饰器）

```python
@router.post("/register", response_model=UserAuthResponse)
async def register(...):
    ...
```

`response_model` 是 **FastAPI 路由装饰器的参数**，作用是自动序列化 + 字段过滤：

```python
class UserOut(BaseModel):
    id: int
    username: str
    # 没有 password！

@router.get("/user/{id}", response_model=UserOut)
async def get_user(id: int):
    user = db.query(User).get(id)  # user 有 password 字段
    return user  # 直接返回 ORM，FastAPI 自动转
```

FastAPI 自动：
1. 用 `from_attributes=True` 读取属性
2. 只保留 `UserOut` 中声明的字段（password 被过滤）
3. 序列化成 JSON 返回

### response_model vs 手动 success_response

| | `response_model` | `success_response`（本项目） |
|---|---|---|
| 属于谁 | FastAPI | 自己写的工具函数 |
| 作用 | 自动序列化 + 过滤字段 | 统一包一层 `{code, message, data}` |
| 返回值 | Pydantic 模型 / ORM 对象 | 已组装好的 `JSONResponse` |

两者不冲突，可以同时使用。本项目选择手动控制（返回 `JSONResponse`），所以没写 `response_model`。

## 九、本项目完整示例

### schemas/users.py

```python
class UserInfoResponse(UserInfoBase):
    id: int
    username: str
    model_config = ConfigDict(from_attributes=True)


class UserAuthResponse(BaseModel):
    token: str
    user_info: UserInfoResponse = Field(..., alias="userInfo")
    model_config = ConfigDict(
        populate_by_name=True,
        from_attributes=True,
    )
```

### routers/users.py

```python
@router.post("/register")
async def register(user_data: UserRequest, db: AsyncSession = Depends(get_db)):
    existing_user = await users.get_user_by_username(db, user_data.username)
    if existing_user:
        raise HTTPException(status_code=400, detail="用户已存在")

    user = await users.create_user(db, user_data)
    token = await users.create_token(db, user.id)

    response_data = UserAuthResponse(
        token=token,
        user_info=UserInfoResponse.model_validate(user),
    )
    return success_response(message="注册成功", data=response_data)
```
