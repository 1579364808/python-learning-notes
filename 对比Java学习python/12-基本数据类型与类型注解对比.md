# 基本数据类型与类型注解对比

## 一句话定性

Python 的类型注解（`name: str`）相当于 Java 的类型声明，但 Python **运行时不会强制检查**，只是给开发者和工具看的"提示"。

## Python 常用类型注解

```python
# ===== 基本类型 =====
name: str = "Alice"           # 字符串
age: int = 25                 # 整数
price: float = 19.99          # 浮点数
is_active: bool = True        # 布尔值
nothing: None = None          # 空值

# ===== 容器类型（需要从 typing 导入）=====
from typing import List, Dict, Tuple, Set, Optional, Union, Any

names: List[str] = ["Alice", "Bob"]           # 列表，元素类型 str
scores: Dict[str, int] = {"math": 90}         # 字典，键 str → 值 int
point: Tuple[int, int] = (10, 20)             # 元组，两个 int
tags: Set[str] = {"python", "java"}           # 集合，元素类型 str

# ===== 可选与联合类型 =====
maybe_name: Optional[str] = None    # 等价于 Union[str, None]，可能是 str 或 None
id_or_name: Union[int, str] = 123   # 可以是 int 或 str

# ===== 任意类型 =====
anything: Any = "随便什么类型"       # 关闭类型检查

# ===== 在 FastAPI 中 =====
@app.get("/hello/{name}")
async def say_hello(name: str) -> dict:   # → dict 是返回值类型
    return {"message": f"Hello {name}"}
```

## Java 对比

```python
# Python 类型注解
def greet(name: str, age: int) -> str:
    return f"{name} is {age}"
```

```java
// Java 类型声明
public String greet(String name, int age) {
    return name + " is " + age;
}
```

## 差异表格

| 对比项 | Python | Java |
|--------|--------|------|
| 声明方式 | `name: str` 冒号后面写类型 | `String name` 类型在前面 |
| 运行时检查 | ❌ 不检查，只是提示 | ✅ 编译器强制检查 |
| 容器类型 | `List[str]`（从 typing 导入） | `List<String>`（泛型） |
| 可选值 | `Optional[str]` 或 `str \| None` | `@Nullable String` |
| 联合类型 | `Union[int, str]` 或 `int \| str` | 无原生支持 |
| 无返回值 | `-> None` 或省略 | `void` |
| 任意类型 | `Any` | `Object` |

## 要点说明

1. **Python 类型注解是"装饰性"的**：写错了也不会报错，但 IDE、mypy、FastAPI 数据验证会利用它。
2. **FastAPI 利用类型注解做数据验证**：你写 `name: str`，FastAPI 自动把路径参数转成 str；写 `age: int`，自动转成 int，传"abc"会返回 422 验证错误。
3. **Python 3.10+ 可以用 `X | Y` 代替 `Union[X, Y]`**：比如 `str | None` 等价于 `Optional[str]`。
