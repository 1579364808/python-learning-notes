# 类属性/字段定义对比：Java vs Python

## Java：必须先定义字段

```java
public class Client {
    // 必须先在类体中声明所有字段
    private String apiKey;
    private String baseUrl;
    private int timeout;

    // 然后在构造函数中赋值
    public Client(String apiKey, String baseUrl, int timeout) {
        this.apiKey = apiKey;
        this.baseUrl = baseUrl;
        this.timeout = timeout;
    }
}
```

## Python：不需要定义，直接赋值即创建

```python
class Client:
    def __init__(self, api_key=None, base_url=None, timeout=None):
        # 没有预先声明，直接 self.xxx = ... 就创建了属性
        self.api_key = api_key
        self.base_url = base_url
        self.timeout = timeout
```

## 核心区别

| 对比项  | Java                          | Python                     |
| ---- | ----------------------------- | -------------------------- |
| 字段声明 | 必须在类体中显式声明类型和名称               | 无需声明，直接 `self.属性名 = 值` 即创建 |
| 类型约束 | 编译时强制类型检查                     | 动态类型，运行时决定                 |
| 可见性  | `private/protected/public` 控制 | 全靠命名约定（`_` 开头表示私有）         |
| 内存分配 | 在声明时就确定了对象内存布局                | 属性动态绑定，每个对象可以有不同的属性集       |

## 深入理解

Python 的对象本质上是**字典**（`__dict__`），属性就是字典的键值对：

```python
class Client:
    def __init__(self, api_key):
        self.api_key = api_key
        self.base_url = "https://default.url"

c = Client("abc123")
print(c.__dict__)  # {'api_key': 'abc123', 'base_url': 'https://default.url'}

# 甚至可以动态添加新属性（Java做不到）
c.new_attr = "hello"
print(c.__dict__)  # {'api_key': 'abc123', 'base_url': 'https://default.url', 'new_attr': 'hello'}
```

Java 的对象在编译时就确定了结构，不能运行时添加新字段。Python 则灵活得多 —— **实例属性在第一次赋值时被动态创建**，这也是你不写 `__init__` 直接 `self.xxx` 就能用的原因。
