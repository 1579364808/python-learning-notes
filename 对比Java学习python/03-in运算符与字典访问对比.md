## `in` 运算符与字典访问对比

### 一句话定性
Python 的 `key in dict` 相当于 Java 的 `map.containsKey(key)`，都是检查键是否存在。

### 代码对比

```python
# Python 写法
# 判断键是否存在
if "knowledge_graph" in results:
    # 链式访问字典
    return results["knowledge_graph"]["description"]
```

```java
// Java 对应写法
// 判断键是否存在
if (results.containsKey("knowledge_graph")) {
    Map<String, Object> kg = (Map<String, Object>) results.get("knowledge_graph");
    if (kg.containsKey("description")) {
        return (String) kg.get("description");
    }
}
```

### 差异表格

| 对比项 | Python | Java |
|--------|--------|------|
| 判断 key 是否存在 | `key in dict` | `map.containsKey(key)` |
| 取值 | `dict[key]` | `map.get(key)` |
| 链式访问 | `dict["a"]["b"]` | `((Map)map.get("a")).get("b")` |
| 键不存在时 | 抛 `KeyError` | 抛 `NullPointerException`（get 返回 null 后调用出错）|

### 要点说明

1. Python 的 `in` 只检查**键**，不检查值——相当于 Java 的 `containsKey()`，不是 `containsValue()`
2. 链式访问 `dict["a"]["b"]` 等同于 Java 的拆箱嵌套 Map 逐层 get
3. 安全写法：每次访问前先用 `in` 判断，否则会抛异常
