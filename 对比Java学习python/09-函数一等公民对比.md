## 函数是一等公民对比

### 一句话定性
Python 的函数是"一等公民"——可以像变量一样赋值、存字典、传参数、从函数返回；Java 的方法必须依附于对象或类。

### 代码对比

```python
# Python：函数可以直接赋值、存字典、传递
def search(query: str) -> str:
    return f"搜索: {query}"

# 赋值给变量
my_func = search
result = my_func("华为手机")  # 调用

# 存到字典
registry = {"Search": search}
func = registry["Search"]
result = func("华为手机")

# 作为参数传递
def execute(func, input):
    return func(input)

execute(search, "华为手机")
```

```java
// Java：方法不能独立存在，需要函数式接口包装
public class Tools {
    // 方法必须属于类
    public String search(String query) {
        return "搜索: " + query;
    }
}

// Java 8+ 用方法引用或 Lambda（本质也是对象）
Map<String, Function<String, String>> registry = new HashMap<>();
registry.put("Search", this::search);  // 方法引用

Function<String, String> func = registry.get("Search");
String result = func.apply("华为手机");

// 作为参数传递
public String execute(Function<String, String> func, String input) {
    return func.apply(input);
}
execute(this::search, "华为手机");
```

### 差异表格

| 对比项 | Python | Java |
|--------|--------|------|
| 函数/方法地位 | 一等公民，独立存在 | 二等公民，必须依附类或对象 |
| 赋值给变量 | `f = func` | `Function<> f = this::method` |
| 存到集合 | `dict["key"] = func` | `map.put("key", this::method)` |
| 调用方式 | `func(args)` | `func.apply(args)` |
| 传递匿名函数 | `lambda x: x+1` | `x -> x + 1` |

### 要点说明

1. **Python 函数就是对象**：`print(type(search))` → `<class 'function'>`，和 `str`、`int` 一样是类型
2. **Java 方法不是对象**：需要 `@FunctionalInterface` 接口（如 `Function`、`Consumer`）作为容器
3. **Python 调用不用 `.apply()`**：直接 `func(x)`，Java 必须 `func.apply(x)`
4. **两者 Lambda 都支持**：Python `lambda x: x+1` 对应 Java `(x) -> x + 1`
