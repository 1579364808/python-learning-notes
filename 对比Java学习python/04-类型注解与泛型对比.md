# 类型注解对比：Java vs Python

## 代码对比

```python
# Python 类型注解（3.9+）
data: list[dict[str, str]]

# Python 3.8 及更早（需要从 typing 导入）
from typing import List, Dict
data: List[Dict[str, str]]
```

```java
// Java 泛型
List<Map<String, String>> data;
```

## 差异表格

| 对比项 | Python | Java |
|--------|--------|------|
| 本质 | 注解，**运行时不起作用**，只是提示 | 编译时强制检查 |
| 写法 | `list[dict[str, str]]` | `List<Map<String, String>>` |
| 嵌套泛型 | `dict[str, str]` 表示键是 str，值也是 str | `Map<String, String>` |
| 不遵守 | 可以往 list 里放 int，不会报错 | 编译报错 |
| 版本 | Python 3.9+ 直接用内置类型 | 从 Java 5 开始 |

## 逐层拆解

```
List[Dict[str, str]]
│    └──┬─── ┘
│      └── Dict[str, str] = 字典，键是字符串，值是字符串
│                           相当于 Java 的 Map<String, String>
│
└── List[...] = 列表，里面每个元素是 Dict[str, str]
                相当于 Java 的 List<Map<String, String>>
```

## 要点说明

1. **Python 类型注解是"贴标签"** —— 写错了代码也能跑，不像 Java 泛型写错编译不通过
2. **Python 3.9+ 可以直接用 `list`、`dict` 等小写内置类型**；老版本要从 `typing` 导入 `List`、`Dict`
3. **`Dict[str, str]`** 第一个参数是键类型，第二个是值类型。`Dict[str, int]` 就是键为字符串、值为整数

## 为什么大写变小写

| 版本 | 写法 | 原因 |
|------|------|------|
| Python 3.5 ~ 3.8 | `from typing import List, Dict` | `list`、`dict` 等内置类型当时不支持 `[]` 操作，所以 typing 模块提供了大写版本 |
| Python 3.9+ | `list[dict[str, str]]` | PEP 585 — 给内置类型加上了 `[]` 支持，就不需要 typing 里那套了 |

## 现在推荐用哪种？

**Python 3.9+ 一律用小写**，大写写法已标记为 deprecated（不推荐）。

两种其实都还能用，但混用不好看：
```python
# ❌ 混用，别扭
data: List[dict[str, str]]

# ✅ 3.9+ 统一小写
data: list[dict[str, str]]

# ✅ 老项目兼容旧版才用大写
data: List[Dict[str, str]]
```

类比 Java：相当于 Java 10 有了 `var` 类型推断，老写法还能用，但新项目都写 `var` 了。
